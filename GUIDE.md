# 📘 Руководство для студента

## Добро пожаловать в курс по React Hook Form!

Этот курс поможет вам освоить одну из самых популярных библиотек для работы с формами в React.

---

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
npm install
```

### 2. Запуск dev-сервера

```bash
npm run dev
```

Откройте браузер по адресу `http://localhost:5173`

### 3. Сборка проекта

```bash
npm run build
```

---

## 📚 Как работать с курсом

### Структура уровней

Каждый уровень содержит:
- **README.md** — теория и примеры кода
- **task.md** — подробные задания
- **Template.tsx** — шаблон для начала работы (с подсказками)
- **Solution.tsx** — готовое решение (смотрите после выполнения!)
- **index.ts** — экспорты

### Рекомендуемый порядок работы

1. **Прочитайте теорию** в `README.md` соответствующего уровня
2. **Изучите задания** в `task.md`
3. **Откройте `Template.tsx`** и следуйте инструкциям TODO
4. **Скопируйте шаблон** в свой файл и начните писать код
5. **Проверьте результат** в браузере
6. **Сравните с решением** в `Solution.tsx`

## 🎯 Программа курса

### Уровень 0: Setup
- Настройка проекта
- Первая форма (Hello World)

### Уровень 1: Основы
- `useForm`, `register`, `handleSubmit`
- Разные типы полей
- `formState` (errors, isValid, isSubmitting)
- `watch`, `setValue`, `getValues`

### Уровень 2: Валидация
- Built-in валидация (required, minLength, maxLength)
- Patterns (email, phone, password)
- Custom validate функции
- Cross-field валидация

### Уровень 3: Валидация по схемам
- Zod базовая
- Yup базовая
- Сложные схемы (объекты, массивы, enum)
- Кастомные сообщения и refine
- Сравнение Zod vs Yup

### Уровень 4: Сложные поля
- Controller для UI-компонентов
- Radio и Select
- Checkbox (одиночный и множественный)
- File upload с валидацией
- Дата и время

### Уровень 5: Динамические формы
- `useFieldArray` (add/remove поля)
- Условные поля
- Зависимые поля (country → city)
- Wizard (multi-step форма)

### Уровень 6: Состояния и UX
- Dirty/Touched states
- Default values и reset
- Focus management при ошибках
- Accessibility (ARIA)
- Performance оптимизация

### Уровень 7: Асинхронность
- Async валидация (проверка на сервере)
- Загрузка данных (edit mode)
- Submit с loading/error
- Debounce для автосохранения

### Уровень 8: Продвинутые техники
- Интеграция с UI-библиотекой
- Кастомные хуки (useFormPersist)
- FormContext (разделение форм)
- localStorage persistence
- **Финальный проект**: комплексная форма регистрации
