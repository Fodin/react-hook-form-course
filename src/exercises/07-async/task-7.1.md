# Задание 7.1: Async валидация

## Цель

Научиться выполнять асинхронную валидацию полей формы.

## Требования

Создайте форму регистрации:

1. Поле username с проверкой на сервере (имитация)
2. Поле email с валидацией формата
3. При потере фокуса на username — проверка доступности
4. Отображение статуса: "Проверка..." / "✅ Доступно" / "❌ Занято"

## Подсказка

Имитация API для проверки доступности:

```typescript
const checkUsername = async (username: string): Promise<boolean> => {
  await new Promise(r => setTimeout(r, 500))
  return !['admin', 'user', 'test'].includes(username.toLowerCase())
}
```

## Чеклист

- [ ] Поле username отправляет асинхронный запрос при потере фокуса
- [ ] Во время проверки отображается "Проверка..."
- [ ] После проверки отображается "✅ Доступно" или "❌ Занято"
- [ ] Поле email валидируется по формату
- [ ] Форма не отправляется, если username занят

## Как проверить себя

1. Введите "admin" в поле username и уберите фокус — должно появиться "❌ Занято"
2. Введите уникальное имя — должно появиться "✅ Доступно"
3. Введите некорректный email — должна появиться ошибка валидации
4. Попробуйте отправить форму с занятым username — отправка не должна пройти
