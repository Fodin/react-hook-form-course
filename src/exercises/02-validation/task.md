1. **Откройте `Template.tsx`** — там находятся все задания уровня

## Задание 2.1: Built-in валидация

### Цель
Научиться использовать встроенные правила валидации.

### Требования
Создайте форму регистрации с валидацией:
1. `username` — 3-20 символов, обязательно
2. `email` — email формат, обязательно
3. `age` — от 18 до 120, обязательно
4. `password` — минимум 6 символов, обязательно

Отображайте ошибки под каждым полем.

---

## Задание 2.2: Pattern валидация

### Цель
Научиться использовать RegExp паттерны.

### Требования
Создайте форму с полями:
1. `phone` — российский номер (+7 XXX XXX-XX-XX)
2. `website` — URL (https://...)
3. `hexColor` — HEX цвет (#FFF или #FFFFFF)
4. `slug` — URL-friendly строка (только латиница, цифры, дефис)

Используйте `pattern` для валидации каждого поля.

### Подсказка с паттернами

```typescript
const patterns = {
  phone: /^\+7\s?\(?\d{3}\)?\s?\d{3}-?\d{2}-?\d{2}$/,
  url: /^https?:\/\/.+\..+$/,
  hex: /^#([A-Fa-f0-9]{6}|[A-Fa-f0-9]{3})$/,
  slug: /^[a-z0-9]+(?:-[a-z0-9]+)*$/,
}
```

## Задание 2.3: Custom валидация с validate

### Цель
Научиться создавать кастомные правила валидации.

### Требования
Создайте форму регистрации с паролем, который должен соответствовать правилам:
1. Минимум 8 символов
2. Хотя бы одна заглавная буква
3. Хотя бы одна цифра
4. Хотя бы один специальный символ (!@#$%^&*)

Используйте объект `validate` с несколькими функциями.

---

## Задание 2.4: Cross-field валидация

### Цель
Научиться валидировать несколько полей вместе.

### Требования
Создайте форму смены пароля:
1. `currentPassword` — текущий пароль (обязательно)
2. `newPassword` — новый пароль (минимум 8 символов)
3. `confirmPassword` — подтверждение (должно совпадать с newPassword)
4. `email` — для уведомления (обязательно)

Добавьте валидацию:
- `confirmPassword` должен совпадать с `newPassword`
- `newPassword` не должен совпадать с `currentPassword`
