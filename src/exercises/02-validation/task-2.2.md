# Задание 2.2: Pattern валидация

## Цель

Научиться использовать RegExp паттерны для валидации полей формы.

## Требования

Создайте форму с полями:

1. `phone` — российский номер (+7 XXX XXX-XX-XX)
2. `website` — URL (https://...)
3. `hexColor` — HEX цвет (#FFF или #FFFFFF)
4. `slug` — URL-friendly строка (только латиница, цифры, дефис)

Используйте `pattern` для валидации каждого поля.

## Подсказка

```typescript
const patterns = {
  phone: /^\+7\s?\(?\d{3}\)?\s?\d{3}-?\d{2}-?\d{2}$/,
  url: /^https?:\/\/.+\..+$/,
  hex: /^#([A-Fa-f0-9]{6}|[A-Fa-f0-9]{3})$/,
  slug: /^[a-z0-9]+(?:-[a-z0-9]+)*$/,
}
```

## Чеклист

- [ ] Все 4 поля присутствуют
- [ ] `phone` принимает формат +7 XXX XXX-XX-XX
- [ ] `website` принимает только URL с http/https
- [ ] `hexColor` принимает #FFF и #FFFFFF, отклоняет #GGG
- [ ] `slug` принимает `my-slug-123`, отклоняет `My Slug!`
- [ ] Ошибки отображаются при некорректном вводе

## Как проверить себя

1. Введите `+7 999 123-45-67` в phone — ок
2. Введите `12345` в phone — ошибка
3. Введите `https://example.com` в website — ок
4. Введите `not-a-url` в website — ошибка
5. Введите `#ff00aa` в hexColor — ок
6. Введите `#xyz` в hexColor — ошибка
7. Введите `my-cool-slug` в slug — ок
