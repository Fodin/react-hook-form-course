# Задание 6.6: setFocus, resetField и getFieldState

## Цель

Научиться использовать продвинутые методы управления полями формы: программный фокус, сброс отдельных полей и получение состояния поля.

## Требования

Создайте форму редактирования профиля:

1. Поля: `name`, `email`, `phone` (все обязательные)
2. При монтировании — автофокус на поле `name` через `setFocus`
3. При неудачном submit — фокус на первое поле с ошибкой через `onInvalid` callback
4. Рядом с каждым полем кнопка "Сбросить" — сбрасывает это поле через `resetField`
5. Под каждым полем показывать статус через `getFieldState`:
   - "Изменено" если `isDirty`
   - "Затронуто" если `isTouched`
6. Используйте `delayError: 500` для плавного появления ошибок

## Интерфейс данных

```typescript
interface ProfileForm {
  name: string
  email: string
  phone: string
}
```

## Чеклист

- [ ] Форма с тремя обязательными полями: `name`, `email`, `phone`
- [ ] При загрузке фокус автоматически на поле `name`
- [ ] При submit с ошибками фокус переходит на первое невалидное поле
- [ ] Кнопка "Сбросить" рядом с каждым полем сбрасывает только это поле
- [ ] Под каждым полем отображаются статусы isDirty и isTouched
- [ ] Ошибки появляются с задержкой 500ms

## Как проверить себя

1. Обновите страницу — фокус должен быть на поле `name`
2. Измените значение поля — статус "Изменено" должен появиться
3. Нажмите кнопку "Сбросить" у поля — значение и статусы должны сброситься
4. Отправьте пустую форму — фокус перейдёт на первое пустое поле, ошибки появятся через ~500ms
