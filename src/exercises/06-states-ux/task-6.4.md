# Задание 6.4: Accessibility (ARIA)

## Цель

Научиться создавать доступные формы с корректными ARIA-атрибутами.

## Требования

Создайте форму с ARIA-атрибутами:

1. `aria-invalid` для полей с ошибками
2. `aria-describedby` для связи поля с сообщением об ошибке
3. `role="alert"` для сообщений об ошибках
4. `noValidate` на элементе `<form>`

## Чеклист

- [ ] Поля получают `aria-invalid="true"` при наличии ошибки
- [ ] Каждое поле связано с ошибкой через `aria-describedby`
- [ ] Сообщения об ошибках имеют `role="alert"`
- [ ] Форма имеет атрибут `noValidate`
- [ ] При отсутствии ошибки `aria-invalid` не установлен или равен `false`

## Как проверить себя

1. Откройте DevTools и проверьте атрибуты полей до и после submit
2. Отправьте пустую форму — поля с ошибками должны получить `aria-invalid="true"`
3. Убедитесь, что `id` сообщения об ошибке совпадает со значением `aria-describedby` поля
4. Проверьте, что скринридер озвучивает ошибки (элементы с `role="alert"`)
