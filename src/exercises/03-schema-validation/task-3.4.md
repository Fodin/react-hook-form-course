# Задание 3.4: Кастомные сообщения и refine

## Цель

Научиться создавать кастомные сообщения об ошибках и использовать `.refine()` для сложных проверок.

## Требования

Создайте форму смены пароля с Zod/Yup:

1. `currentPassword` — текущий пароль (обязательно)
2. `newPassword` — новый пароль (минимум 8 символов, заглавная буква, цифра)
3. `confirmPassword` — подтверждение

Используйте `.refine()` для:

- Проверки что `confirmPassword` совпадает с `newPassword`
- Проверки что `newPassword` отличается от `currentPassword`

Все ошибки должны иметь понятные кастомные сообщения на русском языке.

## Чеклист

- [ ] Все 3 поля присутствуют
- [ ] `.refine()` используется для кросс-полевых проверок
- [ ] Сообщения об ошибках кастомные и понятные
- [ ] Ошибки привязаны к правильным полям (не к форме целиком)

## Как проверить себя

1. Введите разные значения в `newPassword` и `confirmPassword` — ошибка с понятным сообщением
2. Введите одинаковые значения в `currentPassword` и `newPassword` — ошибка
3. Введите слабый `newPassword` (без цифры или заглавной) — ошибка
4. Заполните все корректно — форма отправляется
